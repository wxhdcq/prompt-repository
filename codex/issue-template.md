# GitHub Issue 模板

## 任务目标

【一句话说明要完成什么】

## 背景

【说明为什么要做、当前问题是什么、已有材料在哪里】

## 修改范围

- 【允许修改的文件或目录】
- 【如果不确定，先扫描项目结构再决定】

## 具体步骤

- [ ] 检查现有项目结构和 README。
- [ ] 找到相关入口文件、配置文件和数据文件。
- [ ] 按最小改动完成任务。
- [ ] 更新必要文档。
- [ ] 运行验证命令。
- [ ] 正常 git commit。

## 验收标准

- [ ] 页面、脚本、文档或生成物能按预期使用。
- [ ] README 或使用说明保持准确。
- [ ] 中文显示正常。
- [ ] 没有明显报错。
- [ ] 没有提交敏感信息。
- [ ] 验证命令已运行并记录结果。

## 禁止事项

- 不要提交密钥、token、账号密码、验证码或身份证件。
- 不要伪造数据。
- 不要绕过登录、验证码、付费墙或平台限制。
- 不要重构无关代码。
- 不要删除用户已有内容。
- 不要自动下单或执行金融交易。

## 最终输出

- 修改了哪些文件。
- 怎么验证。
- commit hash。
- 部署链接或本地运行方式。
- 仍然存在的风险或需要人工确认的内容。
