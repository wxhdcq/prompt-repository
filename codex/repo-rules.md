# 仓库规则

- 默认使用中文。
- 内容以 Markdown 为主。
- 每个模板必须能直接复制使用。
- 不存敏感信息，包括密码、验证码、token、身份证件、银行卡号和真实账号凭据。
- 不提交 token、密钥、环境变量实值、本地数据库、日志和缓存。
- 不伪造数据；缺数据时写「数据缺失」或「需二次验证」。
- 不自动交易、不自动下单、不连接券商账户执行交易。
- 不绕过登录、验证码、付费墙、平台限制或风控机制。
- 不用 API fallback 代替正常 git commit/push。
- 每次修改后要说明验证方式。
- 修改应尽量小而清楚，避免无关重构。
- 投资、银行、价格、新闻和法律相关内容必须提醒二次验证。
