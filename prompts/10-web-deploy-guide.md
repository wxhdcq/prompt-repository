# 网页部署教程模板

## 用途

用于 Cloudflare Pages、Vercel、个人主页、HTML 项目部署、GitHub Pages 和网页打不开排查。

## 适用场景

- 想把 HTML、Next.js、React 或静态网站部署到线上。
- 想让别人能打开网页、绑定域名或修复打不开问题。
- 想知道 GitHub、Cloudflare Pages、Vercel 的具体设置。
- 想让 Codex 先检查项目结构，再做最小修复。

## 完整提示词

```text
请帮我把这个网页/项目部署到线上，或者给我完整部署教程。

项目情况：
【填写：HTML 单文件 / Next.js / React / 静态网站 / Cloudflare Pages / Vercel / GitHub Pages】

我的目标：
【填写：让别人能打开 / 绑定域名 / 手机适配 / 修改个人介绍页 / 修复打不开】

请输出：
1. 最简单可行方案
2. 需要准备什么
3. 每一步具体怎么点
4. GitHub 怎么上传
5. Cloudflare Pages 或 Vercel 怎么连接
6. 如何设置构建命令
7. 如何设置输出目录
8. 如何绑定域名
9. 如何检查部署成功
10. 常见报错怎么修

如果是给 Codex 执行：
- 请先检查项目结构
- 不要乱改页面设计
- 优先最小修复
- 确保手机端能看
- 修完后给部署说明
- 正常 git commit/push
```

## 输出格式

- 先给最简单可行方案。
- 再给准备事项和逐步操作。
- 最后给部署成功检查方法、常见报错和 Codex 执行要求。

## 注意事项

- 不要编造部署结果、链接、命令输出或线上状态。
- 重要信息需要二次验证。
- 不要提交密钥、token、账号密码或环境变量实值。
- 部署公开页面前确认不包含私人内容。
- 不要乱改页面设计，优先最小修复。
- 构建命令、输出目录和域名设置需要按项目实际情况确认。
