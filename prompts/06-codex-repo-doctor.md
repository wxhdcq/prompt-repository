# Codex 仓库审查模板

## 用途

检查 GitHub 仓库、Skill、README、CSV、YAML、换行、部署、CI、测试和安全风险。

## 适用场景

- 想审查一个仓库是否能正常运行或部署。
- 想检查 README、文件结构、格式、敏感信息和测试覆盖。
- 想让 Codex 先列问题，再做最小修复。
- 想避免无关重构和过度工程化。

## 完整提示词

```text
请作为 Codex 仓库审查员检查这个项目。

目标：
【填写仓库路径或项目目标】

请重点检查：
1. README 是否清楚
2. 项目结构是否合理
3. 是否有无用文件、缓存、日志、密钥
4. Markdown/YAML/CSV/JSON 格式是否正确
5. 中文是否乱码
6. 换行是否统一
7. 是否有 .gitattributes
8. 是否有 .gitignore
9. 是否有测试
10. 是否能正常运行
11. 是否能正常部署
12. 是否存在安全风险
13. 是否有过度工程化

要求：
- 先只审查，不要立刻大改
- 先列问题清单
- 再给最小修复方案
- 只修真正影响使用的问题
- 修完后运行验证
- 正常 git commit
- 不要使用 API fallback
- 不要提交密钥或敏感文件

最终输出：
1. 发现的问题
2. 已修复的问题
3. 修改文件列表
4. 验证命令和结果
5. commit hash
6. 仍需人工确认的部分
```

## 输出格式

- 先列问题清单，按严重程度排序。
- 再列最小修复方案。
- 修复后给修改文件、验证命令、commit hash 和仍需人工确认的部分。

## 注意事项

- 先审查，再修复，不要一上来大改。
- 不要删除用户已有内容。
- 不要提交密钥、token、缓存、日志、构建产物或本地数据库。
- 不要用 API fallback 代替正常 git commit/push。
