# Codex 任务模板

## 用途

把普通需求整理成可以直接丢给 Codex 执行的任务，减少来回补充和误改范围。

## 适用场景

- 想让 Codex 修改代码、整理仓库、修复页面或生成文档。
- 原始需求比较散，需要变成可执行 checklist。
- 想提前写清修改范围、验收标准和禁止事项。
- 想要求 Codex 最后说明文件、验证、commit 和风险。

## 完整提示词

```text
请把下面需求整理成一份可以直接交给 Codex 执行的任务说明。

我的需求：
【粘贴需求】

请输出以下结构：

# Codex 任务标题

## 目标
一句话说明要完成什么。

## 背景
说明为什么要做、当前问题是什么。

## 修改范围
列出允许修改的文件或目录。
如果不确定，请要求 Codex 先扫描项目结构再决定。

## 具体任务
用 checklist 写出要完成的步骤。

## 验收标准
说明什么结果算完成，包括：
- 页面能打开
- 测试通过
- 文件生成成功
- README 更新
- 没有明显报错
- 移动端适配
- 中文显示正常

## 禁止事项
必须包括：
- 不要提交密钥、token、账号密码
- 不要伪造数据
- 不要绕过登录
- 不要重构无关代码
- 不要删除用户已有内容
- 不要自动下单或执行金融交易

## 最终输出
要求 Codex 最后输出：
- 改了哪些文件
- 怎么验证
- commit hash
- 部署链接或本地运行方式
- 仍然存在的风险
```

## 输出格式

- 输出一份完整 Codex 任务说明。
- 标题、目标、背景、修改范围、具体任务、验收标准、禁止事项、最终输出都要齐全。
- checklist 要具体，能直接执行。

## 注意事项

- 不要让 Codex 接触密钥、token、账号密码或验证码。
- 不要要求 Codex 伪造数据、绕过登录、自动交易或自动下单。
- 修改范围不明确时，要求 Codex 先扫描项目结构。
- 对部署、公开发布、删除数据、扩展权限等操作要明确确认边界。
